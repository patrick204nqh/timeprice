Recipes

One call that handles any "X then in currency A, how much now in currency B?"
The calculator above is this method, wrapped in a form. Same currency on both sides = inflation. Same year = FX. Different = both.
+ 
require "timeprice"

# Pure inflation — same currency, two years
Timeprice.compare(amount: 100, from: ["USD", "1990"], to: ["USD", "2024"]).amount
# => 240.08

# Pure FX — same year, two currencies
Timeprice.compare(amount: 100, from: ["USD", "2010"], to: ["JPY", "2010"]).amount
# => 8866.0

# Cross-cross — FX at the source date, then inflate in the destination economy
Timeprice.compare(amount: 100, from: ["USD", "2010"], to: ["VND", "2024"]).amount
# => 2_932_417.0
Rails view helper for "today's money"
Drop into app/helpers/ and call from any view.
+ 
# app/helpers/price_helper.rb
module PriceHelper
  def in_todays_money(amount, year:, country: "US")
    today  = Date.today.strftime("%Y-%m")
    result = Timeprice.inflation(amount: amount, from: year, to: today, country: country)
    number_to_currency(result.amount)
  end
end

# in_todays_money(2.50, year: "1955")  =>  "$30.12"

"$X then is $Y today" fun fact
For blog posts, social media, or chat-bot replies.
+ 
def fun_fact(amount, from_year, country: "US")
  to    = Date.today.strftime("%Y-%m")
  r     = Timeprice.inflation(amount: amount, from: from_year.to_s, to: to, country: country)
  "$#{amount} in #{from_year} is $#{r.amount.round(2)} today (#{country})"
end

fun_fact(100, 1990)
# => "$100 in 1990 is $242.09 today (US)"

Methodology & accuracy

Cross-check: for $100 USD from 1990 to 2024, the official BLS calculator agrees with us to within a cent of rounding. Pick a specific date (Jun 15 etc.) and we'll match the BLS monthly answer just as closely.

How it works. Inflation = amount × (CPI[to] / CPI[from]), using the monthly index when both dates have one and falling back to annual otherwise. FX uses daily mid-rates (ECB-style); weekend and holiday lookups walk back up to 7 days to the previous trading day. Compare runs FX at the source date first, then inflates in the destination currency — see the "Compare semantics" note for why that order matters.

Rounding. Intermediate ratios are kept at full Float precision; only the final amount is rounded by the caller. Zero-decimal currencies (JPY, KRW, VND) display without decimals.
